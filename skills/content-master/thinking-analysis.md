# 思维分析 - 详细方法论

## 核心定位

思维分析的任务，是把混沌问题变成可判断、可行动、可复盘的结构。它不只是“给建议”，也不是展示工具名，而是帮助用户看清：

1. 这个问题真正卡在哪里。
2. 应该从哪个入口进入。
3. 需要调用哪些思维方式、分析方法、学科视角或现实原则。
4. 最后能形成什么判断、取舍、行动或验证路径。

本文件是思维分析的主调度器。详细工具箱和范式框架见 [references/cross-disciplinary-thinking-toolbox.md](references/cross-disciplinary-thinking-toolbox.md)。

## 调度哲学

### 先判断要交付什么

不要从“有哪些工具”开始，而要先判断这次分析要交付什么：是重新定义问题、解释机制、判断趋势、评估价值、做出取舍、拆出行动，还是把已经发生的事情复盘成可迁移模型。交付物不同，调用工具的方式就不同。

### 工具不是平级菜单

- **一般思维方式**：改变观察角度，决定“怎么看”。例如抽象建模、系统性思考、贝叶斯思维、供需与边际调节思维、工程化思维。
- **典型分析方法**：推进问题，决定“怎么分析下去”。例如数据分析、复盘、构建统计量、输入输出分析、财务报表分析。
- **学科视角和原则**：做现实校正，提醒“是否漏掉一个底层视角”。例如生态位、信息压缩、身体底层变量、不提前下牌桌。
- **执行工具**：承接行动，不一定属于分析方法。例如 OKR、清单、时间表、仪式、最小循环。

输出时不要把四层混成一堆。先用思维方式确定观察角度，再用分析方法形成判断链，必要时用学科视角校正，最后再把结论转成执行工具。

### 先选主工具，再补链条

每次分析先选一个最能推进问题的主工具，再根据缺口补配套工具。

- **主工具**：决定分析主线，必须讲透。
- **配套工具**：补足变量、证据、量化、取舍或行动。
- **校正工具**：防止主工具误用，例如用现实处境、身体状态、财务风险、生态位校正漂亮理论。
- **异质工具**：用不直觉的工具制造新解释，但只有在它确实带来增量时才使用。

工具是否值得调用，看它能不能带来新增信息：是否定义了新对象，提取了新变量，解释了新机制，提供了证据口径，形成了取舍，或把判断转成行动。如果只是换个名词重复原判断，就不要调用。

### 工具有包含和联动关系

不要把工具理解成彼此孤立的按钮。很多分析方法本身就是思维方式的落地形态，或者是另一个分析方法中的一个模块。

- **数据分析**通常包含问题链、抽象建模、构建统计量、历史复盘和贝叶斯更新。构建统计量是数据分析的关键工具，但不等于数据分析整体。
- **复盘**通常包含归纳总结、因果分析、构建统计量和贝叶斯更新。它把已经发生的事情加工成可迁移模型。
- **构建统计量**来自抽象建模，也服务于数据分析、工程化思维和价值判断。它把模糊描述变成可比较基准。
- **输入输出分析**来自热力学和系统论。它先判断系统是否封闭，再看输入、转换、输出和排熵机制，目标是优化系统而不是批评输出。
- **系统性思考**常常继续落到因果分析和主要矛盾分析，找到关键变量后再交给工程化思维做出来。
- **工程化思维**常常调用模块化拆解、构建统计量、OKR、清单、时间表和不可能三角。OKR 和清单是执行承接，不必硬塞进分析方法列表。
- **供需与边际调节思维**内部包含供需平衡表、边际调节链、职业薪酬定价和需求匹配。它先判断稀缺、失衡、边际供给和边际需求，再看价格、条件、库存、工资、门槛、心智位置和行为如何调整。
- **价值判断**常常组合供需与边际调节思维、财务报表、折现值、杠杆和不可能三角。它们分别处理稀缺和选择、财务真实、时间、放大机制和取舍。

### 问题状态是导航，不是边界

问题状态只是导航，不是对世界的封闭分类。真实问题经常跨状态流动，也可能一开始无法归类。无法归类时，先做开放探索：确定对象、边界、变量、证据、约束和可行动空间，再判断主工具。

## AI 调用框架

AI 最重要的不是记住工具名，而是定位：看出用户真正要处理的对象、缺失的判断、可用的证据、现实约束和最终交付物。工具池很大时，调用端必须像调度器，而不是像目录。

### 1. 先做语义解码

拿到问题后，先在内部完成五个判断：

- **对象**：分析的是人、系统、商品、岗位、组织、行业、资产、信息、关系、身体状态，还是已经发生的事件。
- **缺口**：缺的是定义、变量、机制、证据、价值、取舍、行动、表达，还是复盘。
- **证据状态**：目前是只有感受和故事，已有数据，已有历史案例，还是需要先收集信息。
- **约束**：时间、钱、身体、组织权限、行业周期、关系边界、风险承受力和生态位是什么。
- **交付物**：最后要给出判断、框架、文章结构、行动方案、验证指标，还是继续追问的问题。

如果这五点不清楚，不要急着调用工具。先用问题链思维澄清；若用户需要的是直接推进，就用开放探索先建立对象、边界、变量和可行动空间。

### 2. 根据“缺什么”选主工具

主工具不是看关键词选，而是看它能不能推进最关键的缺口。

| 当前真正缺的东西 | 优先主工具 | 典型补链 | 输出重点 |
| --- | --- | --- | --- |
| 对象和问题不清 | 问题链思维、抽象建模 | 课题分离、第一性原理、批判性思维 | 定义、边界、关键变量、待验证假设 |
| 变量关系和时间变化 | 抽象建模与函数化思维、函数化分析 | 构建统计量、数据分析、贝叶斯更新 | 横轴纵轴、曲线阶段、指标口径、趋势判断 |
| 系统为何反复产生同一结果 | 系统性思考、输入输出分析 | 因果分析、主要矛盾分析、生理学或脑科学视角 | 系统边界、反馈回路、输入输出、主要矛盾 |
| 价格、收入、机会和定价权 | 供需与边际调节思维 | 财务报表、折现值、杠杆、不可能三角 | 供需失衡、边际变量、价格/条件/行为路径 |
| 价值能否被别人选择 | 供需与边际调节思维、语言表达视角 | 营销学、人类学、信息压缩 | 需求层次、心智位置、表达结构、传播阻力 |
| 风险、现金流和长期下注 | 财务报表分析、折现值分析、投资哲学 | 贝叶斯更新、杠杆、不可能三角、处境校正 | 基本盘、现金流、下行风险、非对称机会 |
| 方向有了但做不出来 | 工程化思维、模块化拆解 | 构建统计量、OKR、清单、时间表、主要矛盾 | 交付物、模块、最小抓手、验证指标 |
| 事情已经发生，需要沉淀 | 复盘、数据分析 | 因果分析、构建统计量、贝叶斯更新、系统性思考 | 事实还原、变量、模型、边界、下一轮验证 |
| 学习、情绪、习惯、自控失灵 | 输入输出分析、脑科学/生理学视角 | 系统性思考、工程化思维、心理学视角 | 底层状态、机制层级、最小闭环、恢复路径 |
| 信息太多或表达无效 | 语言表达视角、信息论原则 | 问题链思维、认知心理学、供需匹配子系统 | 中心、层级、顺序、事实故事、必要冗余 |

### 3. 用当前总框架扫描盲区

如果问题很大，或工具选择不确定，就用跨学科总框架做一次盲区扫描。扫描不是把二十六讲全用上，而是看问题是否漏了某个底层维度：

- **秩序与系统**：是否需要输入输出、系统论、因果、主要矛盾，解释为什么默认会变乱或反复。
- **实现与变化**：是否需要工程学把判断做出来，或函数视角判断它在时间中如何变化。
- **人自身**：是否需要脑科学、认知心理学、心理学、生理学、自我管理，解释学习、情绪、习惯、身体和动机。
- **资源与价值**：是否需要经济学、金融学、会计学、投资哲学，解释稀缺、现金流、风险、杠杆和下注。
- **关系与表达**：是否需要社会网络、营销学、语言表达、人类学、信息论，解释机会、选择、共识、传播和信息压缩。
- **长周期与处境**：是否需要复杂性科学、历史学、古代哲学、现代哲学，校正时间、涌现、边界、承受力和生态位。

扫描的目的，是发现“主工具解释不了的剩余问题”。如果没有剩余问题，就不加工具；如果有剩余问题，再补一个最能带来增量的校正视角。

### 4. 组合规则

- **一个主线讲透**：复杂问题也要有一个主工具决定正文主线。
- **补链要有角色**：配套工具负责补变量、证据、量化、价值、行动或现实校正，不能只是换名词。
- **先解释后行动**：如果机制不清，先用系统、因果、供需、数据等工具解释；机制清楚后再用工程化工具落地。
- **先保命再优化**：涉及财务、健康、职业、投资和关系风险时，先检查下行风险、现金流、身体状态和关系边界。
- **允许异质碰撞**：当常规工具只能得到常规答案时，可以用不直觉的学科视角碰撞，但必须说明它带来的新增信息。

## 展开深度

### 快速判断模式

适合用户只要方向或建议。输出“判断 -> 原因 -> 下一步”，少讲工具。

### 结构分析模式

适合一般复杂问题。输出“问题定位 -> 工具选择 -> 分析展开 -> 关键判断 -> 下一步”。

### 深度框架模式

适合重大选择、文章框架、研究问题、商业判断、职业转型、长期复盘。需要调用 reference，形成：

- 问题入口树
- 变量与系统结构
- 价值和风险判断
- 行动路径
- 现实校正
- 可验证指标

## 标准输出结构

```markdown
## 问题定位
[判断这是哪类真实问题，以及为什么从这个入口进入]

## 工具选择
[选择 1-3 个主工具，必要时加 1 个现实校正视角]

## 分析展开
[按工具的问题链展开，不堆砌概念]

## 关键判断
1. [最重要判断]
2. [次重要判断]
3. [风险、取舍或验证方向]

## 下一步
[给出可执行动作、需验证信息，或继续追问的问题]
```

当用户需要文章、报告或笔记，思维分析只负责生成判断框架，之后交给 `note-processing.md` 或 `writing-creation.md` 继续加工。

## 工具定位矩阵

这张矩阵不是给人背的，而是给 AI 调度时做“候选工具生成”。先根据用户问题生成 2-4 个候选工具，再按新增信息筛掉多余工具。

| 真实问题信号 | 首选候选 | 继续追问 |
| --- | --- | --- |
| “我不知道问题是什么” | 问题链、抽象建模、课题分离 | 对象是什么，边界在哪里，谁承担后果 |
| “它为什么总是这样” | 系统性思考、因果分析、主要矛盾 | 哪些变量互相增强，哪个变量牵动最多下游 |
| “它未来会怎么变” | 函数化分析、构建统计量、贝叶斯更新 | 变量关系是什么，处在哪条曲线，证据质量如何 |
| “为什么它值钱/不值钱” | 供需与边际调节思维、折现值 | 需求是否真实，供给是否稀缺，边际变量是谁 |
| “我该不该投/去/做” | 财务报表、折现值、供需与边际调节、不可能三角 | 下行是否可承受，现金流能否撑到兑现，放弃什么 |
| “我知道但做不出来” | 工程化思维、模块化拆解、输入输出 | 交付物是什么，最小抓手在哪里，底层状态是否可用 |
| “别人为什么不理解/不选择” | 供需与边际调节思维、语言表达、信息压缩 | 对方需求是什么，中心是否清楚，心智位置在哪里 |
| “这次经历说明什么” | 复盘、数据分析、因果、贝叶斯 | 原假设是什么，偏差在哪里，下一次验证什么 |
| “我状态不好/失控/拖延” | 输入输出、脑科学、生理学、心理学 | 输入是否劣化，身体是否过载，行为在保护什么 |
| “这个社会/行业为什么这样” | 供需与边际调节、社会网络、历史、复杂性科学 | 结构位置如何，长期周期如何，是否存在涌现和共识 |

## 质量要求

1. **必须有入口判断或开放探索判断**：先说这更像哪类问题；如果无法归类，说明先从对象、边界、变量和证据进入。
2. **必须说明工具为什么适用**：不能只列工具名。
3. **必须形成链条**：定义、变量、机制、价值、行动之间至少要有一条连贯线。
4. **必须有现实校正**：重大问题要回到处境、代价、时间概率、环境和生态位。
5. **必须有下一步**：分析要能转成行动、验证或继续追问。
6. **避免工具堆砌**：宁可讲透一个主工具，也不要把输出写成方法名展览。
