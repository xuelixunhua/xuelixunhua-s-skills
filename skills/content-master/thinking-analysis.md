# 思维分析 - 详细方法论

## 核心工具

### 1️⃣ 思维框架类（帮用户"看清"问题）

- **利益溯源**：分析行为背后的利益动机和权力关系
- **成本分析**：识别显性/隐性成本、机会成本、两面性
- **供需分析**：理解市场结构、稀缺性、竞争格局
- **系统思维**：看到要素连接、因果循环、反馈机制
- **第一性原理**：从基本事实推导，而非基于类比

### 2️⃣ 行动方法类（帮用户"做对"事情）

- **流程拆解/SOP化**：把目标变成可执行的标准化步骤
- **逆向工程**：从好结果倒推制作流程和关键决策点
- **结构化提问**：用BTCF框架（背景-任务-约束-格式）精准提问
- **优先级排序**：用重要-紧急矩阵、ICE评分找到最该做的事
- **复盘**：从过去经验中提炼规律，指导未来
- **模块化拆解**：把领域拆成可复用模块，自由组合创新

### 3️⃣ 综合引导类（帮用户"全面"审视）

- **六顶帽检查清单**：用六个视角（事实/情感/风险/价值/创新/统筹）全面审视决策
- **苏格拉底式提问**：通过反向提问帮用户澄清模糊问题

## 工作流程

### 第一步：判断问题性质

- 想理解/分析某事物 → 思维框架类
- 想做成/解决某事 → 行动方法类
- 想全面审视/澄清 → 综合引导类

### 第二步：选择合适工具

根据问题特征选择1-2个最相关的工具（复杂问题可组合多个）

### 第三步：结构化输出

```
## 🔧 工具选择
[说明选择了什么工具，为什么适合这个问题]

## 📊 初步分析
[使用所选工具的框架进行分析]

## 💡 关键洞察
1. [最重要的发现1]
2. [最重要的发现2]
3. [最重要的发现3]

## 🤔 深化问题
为了帮你更深入思考，我想问：
1. [引导性问题1]
2. [引导性问题2]
3. [引导性问题3]
```

## 工具选择速查表

| 问题关键词           | 推荐工具       |
| -------------------- | -------------- |
| 为什么他/他们这样做  | 利益溯源       |
| 值不值得、要不要     | 成本分析       |
| 市场机会、竞争、定位 | 供需分析       |
| 复杂现象、多因素影响 | 系统思维       |
| 本质是什么、创新方式 | 第一性原理     |
| 如何做、怎么实现     | 流程拆解/SOP化 |
| 学习复刻某个成功案例 | 逆向工程       |
| 不知道怎么问         | 结构化提问     |
| 事情太多不知先做啥   | 优先级排序     |

| 建立内容/知识体系    | 模块化拆解     |
| 重大决策需要全面考虑 | 六顶帽检查清单 |
| 思路混乱需要澄清     | 苏格拉底式提问 |

## 核心原则

1. **混合型交互**：先给初步分析，再通过提问引导深化
2. **工具组合**：复杂问题可组合使用（如：第一性原理→利益溯源→流程拆解）
3. **不确定时**：先用苏格拉底式提问澄清问题
4. **输出简洁**：突出关键洞察，避免冗长理论

## 使用示例

**用户**：为什么我的老板总是反对我的提案？

**分析过程**：

---
## 🔧 工具选择
使用【利益溯源】+【系统思维】
- 利益溯源：分析老板的真实动机
- 系统思维：看到组织中的权力关系和反馈机制

## 📊 初步分析
**利益相关者分析**：
- 老板的显性角色：决策者
- 老板的隐性考量：部门预算、上级评价、风险承担
- 你的提案可能触及：资源分配、责任归属、现有流程改变

**可能的动机**：
1. 风险规避：新提案=不确定性=可能的责任
2. 权力维护：提案改变现有决策流程，削弱其控制感
3. 资源竞争：你的项目可能占用他优先级更高的资源

## 💡 关键洞察
1. 问题可能不在提案本身，而在于老板的利益/风险感知
2. 反对可能是系统性的（组织文化、考核机制），而非针对你个人
3. 需要从"说服老板"转向"降低老板的风险感知"

## 🤔 深化问题
1. 你的提案会让老板承担什么新的责任或风险？
2. 老板的KPI考核指标是什么？你的提案对此有何影响？
3. 有没有成功通过的提案？它们有什么共同特点？
