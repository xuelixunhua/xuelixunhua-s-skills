# 笔记处理 - 详细方法论

## 核心工具

1. **笔记解构**：重构笔记的逻辑和结构
2. **会议记录**：将讨论转化为结构化纪要
3. **知识结晶**：将音频/视频转化为知识文档
4. **深度分析**：提取思维方式和方法论

## 工作流程

1. **内容理解**：仔细阅读输入内容，识别显性观点和隐性逻辑
2. **结构分析**：总结主题思想及行文结构，识别论证路径
3. **优化重构**：思考内容想表达什么，结构及逻辑上如何优化
4. **输出结果**：生成优化后的笔记，既有宏观架构又有微观细节

## 表达结构选择原则

根据内容性质选择最合适的表达方式，避免单一结构：

### 文字段落适用于：
- 概念解释、理论阐述、机制描述
- 多个关联性强且共同构成完整概念的要点
- 因果关系、逻辑推理、背景介绍
- 连贯的叙述性内容

### 列表适用于：
- 要点归纳、步骤流程、并列信息
- 分类枚举、条件罗列
- 需要突出条理性的内容

### 表格适用于：
- 对比分析（多个对象的多个维度）
- 分类整理（类别与特征的对应）
- 数据展示、参数对比
- 多维信息的结构化展示

## 输出要求（核心方法论）

### 1. 结构逻辑
- **清晰的逻辑结构**：核心命题、三段论、因果关系、时间顺序、层次结构、概念定义等
- **合理的段落架构**：概念、观点在前，论据居中，案例故事在后
- **整体架构思路**：减少或消除知识点的割裂和重复，从第一性原理、从原理到实践的角度重构

### 2. 表达质量
- **简洁清楚**：语句清楚、易于理解，避免复杂冗余，照顾读者感受
- **结构多样**：根据内容性质选择最合适的表达结构，不要默认使用单一结构
- **各司其职**：文字段落用于阐述和解释，列表用于归纳和枚举，表格用于对比分析

### 3. 内容完整性
- **必须涵盖**：每个段落的核心观点和知识型细节，保留重要论据、案例
- **逻辑完整**：确保逻辑链条清晰且信息完整
- **优先级**：细节完整性优先于简洁性（宁可多记录，不可有遗漏）

### 4. 分类整理
- **区分性质**：区分观点、论据、案例的不同性质
- **合理安排**：按内在联系合理安排架构
- **提炼升华**：如可能，提炼更高层次的概念、规律或原则

### 5. 严格限制
- **不写思考过程和建议**：直接输出结果
- **不补充原文没有的内容**：保持客观，只整理不扩写

## 分析思路

考虑以下内容：
1. 段落的整体观点和分段观点
2. 段落的逻辑链条
3. 段落的具体论据
4. 识别可对比的信息、可分类的数据、可枚举的要点

## 特殊场景处理

### 课程笔记整理
- **全覆盖原则**：确保课堂提到的所有知识点都被记录
- **逻辑化原则**：按照知识的内在逻辑关系重新组织内容，而非简单按时间顺序
- **层次化原则**：建立清晰的知识层次结构（概念→原理→应用→案例）
- **可复习原则**：便于后续查阅和复习，重点突出，脉络清晰

### 会议记录整理
- **结构化框架**：会议主题、日期时间、参会人员、主要事项、主要讨论、决定和行动计划、下一步打算
- **信息准确性**：不对用户提供的信息做扩写，仅做信息整理
- **宁可多记录，不可有遗漏**

### 知识结晶（音频/视频转文档）
- **零遗漏捕获**：所有有价值信息都要记录
- **构建理解路径**：自然流畅的逻辑结构
- **认知友好**：便于理解和记忆
