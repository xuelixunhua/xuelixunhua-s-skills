# 宏观研究框架

这份参考文件沉淀的是宏观研究和宏观笔记的专属方法。适用于整理宏观研报、维护宏观总笔记、重构宏观章节、补全指标和政策接口等场景。

## 宏观笔记的目标

宏观笔记不是“宏观报告摘要”，而是把后续分析真正要调用的对象、框架、定义、指标、抓手和约束压成可复用结构。

宏观笔记要满足：
- 可复习
- 可调用
- 可转述
- 可跟踪

## 宏观内容的五类信息

宏观材料里常见的信息，先分类型再归位：

| 类型 | 典型内容 | 解决什么问题 |
| --- | --- | --- |
| 观察角度 | 周期、三流体系、供需、总量与结构 | 用什么镜头看 |
| 历史脉络 | 中国增长模式、红利切换、转型约束 | 为什么中国要这样看 |
| 分析对象 | 增长、通胀、政策、流动性 | 核心变量怎么拆 |
| 跟踪方法 | 指标体系、快指标、高频验证、政策时间窗口 | 平时拿什么跟 |
| 延伸解释 | 定义、公式、口径、使用边界、误区 | 某个对象具体怎么用 |

不要把观察角度、历史脉络、分析对象、跟踪方法混在一层。

## 宏观笔记推荐骨架

推荐按下面顺序组织：

1. 宏观研究在看什么
2. 为什么中国宏观要这样看
3. 平时拿什么跟
4. 核心变量怎么拆
5. 宏观如何映射到资产
6. 月度如何跟踪和记录

可对应为：
- 宏观研究定位
- 观察角度
- 中国经济发展脉络
- 指标体系与跟踪抓手
- 增长
- 通胀
- 政策
- 宏观到资产价格
- 月度跟踪怎么记

## 增长部分怎么写

增长部分默认骨架：

1. 常见经济周期
2. GDP
3. 长期增长
4. 就业
5. 需求分解
6. 投资 / 消费 / 外贸
7. 供给验证

关键要求：
- GDP 下放定义、三种核算、名义/实际、平减指数，不要拆散。
- 需求之下可以继续分投资、消费、出口。
- 投资之下可以继续分固投口径、地产、基建、制造业投资。
- 就业、外贸不能遗漏，它们是增长框架的一部分，不是边角补充。

## 通胀部分怎么写

通胀部分至少包含：
- CPI 定义
- PPI 定义
- CPI 的自上而下框架
- CPI 的自下而上分项
- PPI 的特征和原因
- PPI 的供需矩阵
- CPI / PPI 的传导关系

不要只写“CPI 看猪价、PPI 看大宗商品”这类口号式结论。

## 政策部分怎么写

政策部分至少包含：
- 政策总览：财政、货币、监管、产业等如何作用基本面
- 政策周期：增长/通胀/库存/资产的组合关系
- 历史政策：财政、货币、监管放在同一张表里
- 财政：口径、四本账、赤字、广义财政、月度抓手
- 货币与信用：目标、中介锚、工具、社融、M1/M2、双支柱
- 政策跟踪：会议、频率、时间窗口

政策部分优先表格化，因为它天然是多维比较信息。

## 宏观细节默认标准

宏观里“有细节”通常意味着要补下面这些内容：

| 类别 | 至少要补什么 |
| --- | --- |
| 定义类 | 概念定义、统计范围、适用对象 |
| 公式类 | 计算关系、近似关系、不要只写意义 |
| 指标类 | 看什么、怎么用、领先还是同步、局限是什么 |
| 口径类 | 不同口径差异、是否有统计调整 |
| 历史类 | 历史阶段、政策演变、制度变化 |
| 跟踪类 | 发布频率、时间窗口、高频验证 |

## 宏观里最容易漏掉的接口

常见漏项：
- PMI 的定义、计算和分项使用
- GDP 平减指数
- 就业的完整统计接口
- 外贸的结构、运价和口径差异
- 社融的部门和渠道拆解
- M1 / M2 的差别和先后关系
- 财政四本账
- 货币工具解释
- 双支柱
- 政策跟踪时间窗口
- 统计口径调整

## 宏观里的标题原则

- 标题要像正常讲宏观笔记时会说的话。
- 不要因为 PPT 换页就新开标题。
- 真并列才并列，延伸说明就并回主体。
- 内容量很小的小标题要尽量合并。

## 宏观里的表达要求

- 不写“报告说”“报告给出”。
- 不写讲课腔、口语化提示语。
- 简单公式、一般概念不加反引号。
- 多维比较优先表格，不要长列表堆砌。

## 交付前核对清单

- 是否已经区分观察角度、历史脉络、分析对象、跟踪方法。
- 是否所有核心变量都具备定义、抓手、机制、边界。
- 是否存在同一主体被拆散、重复或层级错位。
- 是否遗漏宏观常见接口：就业、外贸、社融、M1/M2、统计调整、政策时间。
- 是否真正写成了笔记，而不是“宏观课件文字版”。
